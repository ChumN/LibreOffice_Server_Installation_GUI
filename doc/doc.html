Documentation

For Version 3.0.x

This tool allows to do Server Installations of LibO without deeper knowledge about the WINDOWS installation process. A Server Installation is independent from the normal LibO Installation and does not interact with that other LibO.

The Server Installation can be used for tests with experimental LibO versions and if you additionally need a particular LibO Version because the feature is still only available in experimental versions or broken in your current normal LibO installation.

The GUI Installer is not necessary for Installation of LibO Master Versions from http://dev-builds.libreoffice.org/daily/master/Win-x86@6/current/ (and similar), these Master Versions will be installed independent from the normal stable LibO version by default. But of course you also can install a Master Version (or stable version) using the GUI Installer, for example if you want that that version will not be overwritten by later (Master) installations.

System Requirements

Windows Installer 4.5 Microsoft .NET Framework 4 Client Profile (x86 und x64)

Installation

You can simply install the tool by a double click on the downloaded setup.exe in the WINDOWS File Explorer. If you doesn't have a prerequisite installed, it automatically will be installed before the installation of LibreOffice Server Install GUI starts

After Installation you will find in the folder "Reisisoft" “LibreOffice Server Install GUI” in the WINDOWS Programs Start Center.

bootstrap.ini

The input pane allows to define what user profile the server installation of LibO will use. Details concerning User Profile you find here:https://wiki.documentfoundation.org/UserProfile#User_profile_location.

There are 4 appropriate places for your user profile:

a) You change nothing, your Server installation will use (and create, if necessary) the default profile defined by the default entry in bootstrap.ini like “UserInstallation=$SYSUSERCONFIG/LOdev/3”

b) You use your normal user profile with input "UserInstallation=$SYSUSERCONFIG/LibreOffice/3" (for LibO Version 3.x). That's risky, your User Profile might become unusable

c) you use a separate User Profile only for this version with entry "UserInstallation=$ORIGIN/../Data/settings"

d) You use a separate common user profile for your experimental Server Installations with entries like "UserInstallation=$SYSUSERCONFIG/LibO-dev/3" or "UserInstallation=$SYSUSERCONFIG/LibreOffice/3experimental" or similar e) You may use the snipped provided right beneath the textbox in order to not override your profile

Manager

Clicking the manager button will open a new manager window. In the manager you can Add or Delete existing parallel installations.

Add

Inside the manager, click "Manually add parallel installation". A dialogue will pop up. Click "Configure path" and select the soffice.exe of the parallel installation. Confirm by clicking Okay.

Delete

Check all list parallel installation you want to delete. After that, click "Delete selected". After the command line disappears, the parallel installation has been finished. Attention: Deleting files is always risky!

Caveat emptor, you do all this on your own risk, but it should be working without any problem...
